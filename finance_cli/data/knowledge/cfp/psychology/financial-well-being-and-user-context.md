# Financial Well-Being and User Context

Financial well-being is the user's lived experience of money: control over
day-to-day finances, capacity to absorb shocks, progress toward goals, and
freedom to make choices. The same balance sheet can feel safe to one user and
fragile to another because income volatility, dependents, past experiences, and
confidence differ.

This topic helps the advisor agent include user context without turning into
therapy or making unsupported psychological claims.

## Context Signals

Capture context that changes planning fit:

- confidence or anxiety about the decision;
- income predictability and employment stability;
- recent shock: job loss, medical bill, move, divorce, death, caregiving, or
  business disruption;
- household responsibilities and dependents;
- attention budget for detailed budgeting or ongoing monitoring;
- prior experience with debt, investing, insurance, taxes, or financial
  professionals;
- user preference for automation, manual review, simplicity, or control.

These signals should influence how a plan is sequenced and explained. They
should not be used to manipulate the user or override stated goals.

## Well-Being Dimensions

Use the CFPB framing as a planning lens:

- day-to-day control;
- ability to absorb financial shocks;
- progress toward goals;
- choices that allow the user to enjoy life.

A plan that maximizes one dimension while damaging another may fail. For
example, aggressive debt payoff can improve long-term progress but harm shock
absorption if it drains all liquidity.

## Practical Application

For `coach_financial_plan_intake`, include user context in the Financial
Planning Snapshot:

- "The user values stability over speed."
- "Income is seasonal, so the liquidity target should be more conservative."
- "The user wants to invest but is anxious about market losses; start with
  education and account-funding boundaries before any implementation."
- "The user reports high stress and missed bills; route to cash-flow and debt
  stabilization before long-horizon planning."

The point is not to score the user. The point is to make the next action fit the
person who has to carry it out.

## Boundaries

The coach may:

- ask about confidence, stress, values, and preferences;
- reflect how those facts affect planning sequence;
- recommend simpler, safer, or more supported workflows;
- suggest outside support when financial stress appears connected to crisis,
  coercion, or mental health risk.

The coach must not:

- diagnose anxiety, depression, trauma, addiction, coercion, or incapacity;
- shame the user for past decisions;
- use stress as a reason to push a financial product;
- present a financial well-being score as a clinical or creditworthiness measure.

## Common Pitfalls

- Treating financial planning as pure math.
- Ignoring income volatility because the current month looks fine.
- Choosing the highest-maintenance plan for a user with low attention budget.
- Treating a user's fear as irrational instead of as planning data.
- Letting emotional support replace concrete next steps.
